# AI Agent Feedback and Improvement Log

## Purpose

This log is used to turn one-time feedback into reusable improvement.

Without this log, the team risks:

- repeating the same mistakes
- depending on temporary context
- improving only accidentally

## Log Format

| Date | Role | Task / Context | Feedback Received | Root Cause | Improvement Action | Asset to Update | Owner | Status |

## Usage Rules

- Record only meaningful feedback
- Focus on reusable lessons, not emotions
- Link the feedback to an actual improvement action
- If a pattern repeats, upgrade a template or checklist

## Typical Improvement Targets

- Role definition
- Messaging template
- Checklist
- Playbook
- Case study structure
- Review standard
