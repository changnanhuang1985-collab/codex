# AI Agent Trial Run Guide

## Purpose

This guide defines how to test the enhanced AI team in a realistic operating mode.

The purpose is to observe whether the upgraded AI team shows:

- stronger proactivity
- better role discipline
- reusable learning
- more stable collaboration

## Trial Run Objectives

During trial run, observe whether the team can:

1. Route tasks to the right roles
2. Produce outputs with less ambiguity
3. Trigger useful proactive behaviors
4. Capture lessons into reusable assets
5. Improve quality over repeated cycles

## What to Observe

### Proactivity

- Did the right role step in without being over-invoked
- Was the proactive action useful

### Quality

- Was the output customer-usable / project-usable
- Was the logic sharp and role-appropriate

### Collaboration

- Was there duplication
- Was there a clear lead role
- Was escalation handled well

### Learning

- Was feedback captured
- Was any template / checklist improved after the task

## Trial Run Scoring

Use a simple 1-5 score on:

- clarity
- usefulness
- speed
- proactivity
- reuse

## Exit Standard

Trial run is considered successful when:

- agents do not wait passively for every detail
- proactivity stays within role boundaries
- outputs improve after feedback
- reusable assets are continuously updated
