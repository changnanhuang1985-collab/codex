# Integration Standards

## 目的

本文档用于定义部门内部 `SAP BTP Integration` 标准，确保项目交付一致性、可维护性和可扩展性。

## 1. 设计标准

- 接口设计前必须明确业务目标、源系统、目标系统和触发方式
- 每个接口必须定义所属集成模式
- 每个接口必须说明错误处理机制
- 每个接口必须说明重试策略
- 每个接口必须说明监控方式

## 2. 开发标准

- `iFlow` 命名需遵循统一规范
- `Groovy Script` 仅用于标准组件无法覆盖的场景
- 优先使用标准步骤，不滥用自定义脚本
- `Message Mapping` 与脚本职责要清晰分离
- 公共逻辑尽量复用，避免重复实现

## 3. 安全标准

- 优先使用企业统一认证方式
- 外部接口必须明确认证机制
- 涉及证书的接口必须记录证书生命周期
- 敏感信息不得硬编码在脚本和文档中
- 权限控制必须符合最小授权原则

## 4. 监控标准

- 每个生产接口必须可监控
- 关键异常必须可告警
- 关键接口必须有排查手册
- 日志保留策略应与项目要求一致

## 5. 文档标准

- 每个接口至少应有一份设计说明
- 核心接口需要补充测试样例
- 项目结束后需要沉淀到知识库
- 文档正文以中文为主，术语可保留英文

## 6. 交付标准

- 上线前必须完成测试记录
- 上线前必须完成部署清单
- 上线后必须明确运维接手方式
- 变更记录必须可追踪
