# 信華信 SAP BTP Integration チーム
## SAP Integration Suite / CPI を中心とした実践的な連携支援体制

### 私たちがご提供する価値

当チームは、`SAP Integration Suite / CPI` を中核として、`SAP - SAP`、`SAP - Non-SAP`、`Cloud - On-Premise` の連携領域を対象に、設計・開発・試験・運用初期支援まで一貫してご支援します。  
単なる `iFlow` 開発要員ではなく、日本向け案件で求められる設計品質、レビュー対応、試験証跡、運用観点を含めた delivery を重視しています。

---

## 1. 当チームの Integration ポジション

### 3名の経験者を核に、CPI 体制を重点強化

- `SAP Integration Suite / CPI` に関する関連経験を持つ 3 名を中心に体制を構築
- 個人経験に依存せず、設計・開発・試験・運用の標準化を推進
- まずは切り出しやすい Integration スコープから、堅実に支援可能

### BTP 全体理解を前提とした連携設計

- `Integration` を単独機能ではなく、`BTP` 全体の中で設計
- 接続だけでなく、品質・可運用性・将来拡張性を考慮
- 日本側チームと協業しやすい役割分担を重視

## 2. 主な対応領域

### 設計

- IF 要件整理
- `iFlow` 設計
- 項目マッピング整理
- エラー処理 / 再処理設計
- 運用・監視観点整理

### 開発

- `Cloud Integration (CPI)` 実装
- `Message Mapping`
- `Value Mapping`
- API / OData / File / SFTP 連携
- `SAP / Non-SAP` 接続

### 品質・運用

- 単体 / 結合試験支援
- 日本語レビュー対応
- `MPL / Trace` を前提とした調査設計
- 運用手順・障害切り分け観点整理

## 3. お客様にとってのメリット

### 安心して切り出せる

大規模な全体統合を一気に任せるのではなく、まずは `CPI` を中心とした切り出しやすい範囲から、安定的にご支援できます。

### 日本向け品質で進められる

日本語での設計書、レビュー指摘対応、試験証跡、運用観点まで含めて、対日案件に必要な品質を意識した進め方を行います。

### 技術だけでなく delivery を重視

連携を「つなぐ」だけでなく、「設計」「テスト」「運用初期支援」まで含めて一連で考えるため、日本側チームの負荷を下げやすい点が特徴です。
