# 日本顧客向け
## 「当社は CPI 交付力が強み」で伝えるための PPT メッセージ整理

## 1. 先に結論

お客様に `SAP BTP` チーム紹介をする際、`BTP` 全般を広く浅く見せるよりも、まずは以下のように伝えた方が刺さりやすい。

### 推奨メッセージ

当社の中核強みは、`SAP Integration Suite`、特に `Cloud Integration (CPI)` を中心とした設計・開発・試験・運用初期支援です。  
単なる `iFlow` 実装要員ではなく、業務要件を踏まえた IF 設計、マッピング、接続方式、監視・障害調査まで、一連で支援できることが特徴です。

つまり、見せ方の軸は以下に絞るべき。

- `BTP 全部できます` ではなく `CPI を主軸に実務で支援できます`
- `機能紹介` ではなく `案件を回せる交付力があります`
- `抽象的な技術力` ではなく `日本向け品質での設計・実装・試験ができます`

## 2. お客様が知りたいこと

日本のお客様、特に `CTC` 類似の `BTP` チームが知りたいのは、製品機能の羅列ではなく、次の点である。

- この会社は `CPI` 案件を本当に回せるのか
- `SAP - Non-SAP` 連携や `S/4HANA` 周辺連携を任せられるのか
- 設計書、レビュー、試験、障害調査まで対応できるのか
- 日本側メンバーの手離れを良くしてくれるのか
- 小中規模案件で安定して一緒に進められるのか

そのため、PPT では「BTP の機能一覧」よりも、「CPI を中心に、どの工程を、どのレベルでできるか」を前面に出すべきである。

## 3. どう見せるべきか

## 3.1 伝え方の基本方針

`BTP` 全体を主役にするのではなく、`CPI` を主役にする。

見せ方のイメージ：

- 主役：`Integration Suite / CPI`
- 補助要素：`API Management`、`Destination`、`OAuth`、`Certificate`
- 周辺説明：必要に応じて `BTP Extension` や `App Dev`

つまり、以下の順番で説明するのがよい。

1. 当社の中核は `CPI` 交付能力
2. `SAP - SAP`、`SAP - Non-SAP`、`Cloud - On-Premise` 連携を支援可能
3. `iFlow` 実装だけでなく、設計、マッピング、試験、障害調査まで対応可能
4. 必要に応じて `API Management` や `BTP` 周辺サービスも組み合わせられる

## 3.2 避けた方がよい見せ方

- `BTP` の機能を横並びで大量に並べる
- `Automation`、`App Dev`、`AI` などを同じ重みで並べる
- `Integration` が一要素に埋もれる
- 実績より機能説明が多い
- `CPI` で何を deliver できるかが見えない

## 4. 顧客に刺さるアピールポイント

PPT では、以下の 5 点を主メッセージにすると強い。

## 4.1 CPI を中心とした設計・開発対応力

言いたいこと：
- `Cloud Integration` を用いた連携設計と `iFlow` 開発が中核強み

推奨表現：
- 当社は `SAP Integration Suite` の中でも、特に `Cloud Integration (CPI)` を中心とした交付実績と設計・実装力を強みとしています
- `iFlow` の開発だけでなく、連携方式整理、項目マッピング、例外設計、試験支援まで一貫して対応可能です

## 4.2 SAP / Non-SAP 連携への対応力

言いたいこと：
- `S/4HANA`、`SuccessFactors`、`Ariba` など SAP 製品だけでなく、外部システム連携も実務として対応可能

推奨表現：
- `SAP - SAP` 連携はもちろん、`SAP - Non-SAP`、`Cloud - On-Premise` 連携にも対応可能です
- API、ファイル、OData、SOAP、SFTP など、案件に応じた接続方式で設計・実装します

## 4.3 日本向け品質での交付力

言いたいこと：
- 日本案件で求められる設計書、レビュー対応、証跡整備まで含めて支援可能

推奨表現：
- 日本語での設計書作成、レビュー指摘対応、試験証跡整理まで含めて支援可能です
- 単なる実装要員ではなく、日本向け品質での交付を意識した進め方を重視しています

## 4.4 運用・障害調査まで含めた実務対応力

言いたいこと：
- 作って終わりではなく、監視・障害調査も考えた `CPI` 設計ができる

推奨表現：
- `MPL`、`Trace`、エラー通知、再処理方針を含めた運用性を考慮した設計が可能です
- 本番後の問い合わせや障害切り分けを見据えたログ・監視設計を重視しています

## 4.5 小中規模案件での現実的な推進力

言いたいこと：
- 大規模すぎる話ではなく、現実的な案件でしっかり成果を出せる

推奨表現：
- 小中規模の `CPI` 案件に対し、設計から試験・ハイパーケアまで高密度で支援できます
- 少人数体制でも、役割分担とレビューを明確にし、品質を担保しながら進めます

## 5. PPT に入れるべき構成

`Integration` 紹介スライドは、以下の 2 ページ構成が分かりやすい。

## Page 1
### 当社の Integration 強み

推奨タイトル：

`当社の強みは SAP Integration Suite / CPI を中心とした実践的な交付力です`

入れる内容：

- `CPI` を中心にした支援範囲
  - 連携方式整理
  - `iFlow` 設計・開発
  - メッセージマッピング
  - API / ファイル / OData / SOAP / SFTP 連携
  - 試験支援
  - 監視・障害調査支援
- 対応システム例
  - `S/4HANA`
  - `SuccessFactors`
  - `Ariba`
  - `SAP / Non-SAP`
  - `Cloud / On-Premise`
- 品質面
  - 日本語設計書
  - レビュー対応
  - 試験証跡
  - 運用考慮

## Page 2
### CPI 案件をどう支援できるか

推奨タイトル：

`設計から試験・運用初期支援まで、一連でご支援可能です`

入れる内容：

- 上流
  - 連携要件整理
  - IF 設計
  - 項目・マッピング定義
- 開発
  - `iFlow` 実装
  - `Groovy Script`
  - `Value Mapping`
  - 接続設定
- 品質
  - 単体・結合試験
  - エラーケース確認
  - レビュー反映
- 運用
  - `MPL / Trace`
  - 障害調査
  - ハイパーケア支援

## 7. 口头介绍时怎么说

如果你在介绍这一页，建议不要讲成“我们会很多 BTP 的东西”，而是讲成下面这样：

### 日语话术示例

当社の `BTP` 対応領域はいくつかありますが、特に強みとしているのは `SAP Integration Suite`、その中でも `Cloud Integration (CPI)` を中心とした交付です。  
具体的には、`iFlow` の実装だけではなく、連携方式の整理、項目マッピング、接続設計、試験、さらには本番後の障害調査まで含めてご支援可能です。  
`S/4HANA`、`SuccessFactors`、`Ariba` をはじめ、`SAP / Non-SAP` 連携にも対応しており、日本向け案件で求められる設計書・レビュー・証跡整備も重視しています。

## 8. 一句话定位

如果要压缩成一句最关键的话，可以用这句：

`当社は、SAP Integration Suite / CPI を主軸に、日本向け品質で設計・開発・試験・運用初期支援まで対応できるパートナーです。`

## 9. 最后建议

这份 PPT 里，`Integration` 部分最重要的不是“看起来技术范围很广”，而是让客户明确感受到下面三点：

- 你们的主战场是 `CPI`
- 你们不是只会开发，而是能交付
- 你们的交付方式适合日本客户

如果这三点能被看清楚，客户更容易把 30 人月左右的项目先切给你们。
