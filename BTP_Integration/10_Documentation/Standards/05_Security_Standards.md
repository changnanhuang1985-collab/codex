# セキュリティ標準

## 1. 目的

本書は `SAP BTP Integration` 領域における認証、認可、通信保護、秘密情報管理の標準を定義する。

## 2. 基本方針

- セキュリティ要件は設計初期に定義する
- 最小権限の原則を適用する
- すべての外部通信は保護されなければならない

## 3. 認証標準

- 原則 `OAuth` を推奨する
- 双方向信頼が必要な場合は `mTLS` を検討する
- パートナ要件に応じて証明書認証、Basic、SAML 等を個別判断する

## 4. 認可標準

- ロール単位でアクセス権を定義する
- 管理者権限の濫用を避ける
- 本番運用権限と開発権限は分離する

## 5. 秘密情報管理

- パスワード、シークレット、トークンを文書やコードに平文で記載しない
- セキュアストアや適切な保管機構を利用する
- 証明書期限を管理台帳で追跡する

## 6. 通信保護

- HTTPS を標準とする
- 外部公開エンドポイントは TLS 設定を確認する
- 証明書の更新責任者と更新タイミングを明確化する

## 7. ログと監査

- 機密データのフル出力は禁止する
- 監査に必要な操作履歴は保持する
- セキュリティイベント発生時の通知先を定義する

## 8. レビュー観点

- 認証方式は妥当か
- 権限分離は適切か
- 機密情報露出はないか
- 証明書管理は定義されているか
- セキュリティインシデント時の運用があるか
