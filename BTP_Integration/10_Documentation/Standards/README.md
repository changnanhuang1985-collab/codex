# SAP BTP Integration 標準・規約ドキュメント一覧

## 目的

本フォルダは、`SAP BTP Integration` プロジェクトにおける設計・開発・運用・品質確保のための標準、規約、制約条件を体系的に整理したものです。

顧客説明、社内標準化、開発ベンダー統制、設計レビュー、品質監査に利用できるよう、実務適用可能な粒度で記載しています。

## ドキュメント一覧

- `01_Architecture_Principles.md`
  - アーキテクチャ原則、責務分離、レイヤリング方針
- `02_Integration_Design_Standards.md`
  - 集成設計標準、インタフェース設計時の基本要件
- `03_iFlow_Development_Standards.md`
  - `Cloud Integration` / `iFlow` 開発標準
- `04_API_Standards.md`
  - `API Management` を含む API 設計・公開標準
- `05_Security_Standards.md`
  - 認証、認可、証明書、機密情報管理などのセキュリティ標準
- `06_Monitoring_Operations_Standards.md`
  - 監視、運用、障害対応、変更管理標準
- `07_Testing_Quality_Standards.md`
  - テスト、品質保証、レビュー基準
- `08_Naming_Conventions_JA.md`
  - 命名規約
- `09_Delivery_Governance_Standards.md`
  - ドキュメント統制、承認、移行、納品管理標準
- `10_Git_CICD_and_AI_Governance_Standards.md`
  - `Git`、`CI/CD`、`AI` 協作使用の統制標準

## 想定利用シーン

- 提案書・方針説明資料のベース
- プロジェクト開始時の標準説明
- 設計レビュー基準の明文化
- ベンダー管理基準
- 顧客向け運用・品質説明資料

## 利用上の考え方

- 本標準は「最低限守るべき基準」を定義する
- プロジェクト特性に応じた追加制約は別紙で定義する
- 例外を認める場合は、理由・影響・代替策・承認者を明記する
