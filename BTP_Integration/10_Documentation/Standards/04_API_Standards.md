# API 標準

## 1. 目的

本書は `SAP API Management` を含む API の設計、公開、保護、ライフサイクル管理を標準化するための規約を定義する。

## 2. 基本方針

- API はシステム内部実装ではなく契約として設計する
- API は利用者視点で設計し、バックエンド都合を露出しすぎない
- セキュリティ、バージョン、利用制御を設計初期から定義する

## 3. API 分類

- `System API`
  - 基幹システム能力の標準化
- `Process API`
  - 複数システムを横断する業務処理の公開
- `Experience API`
  - 利用チャネル別の最適化公開

## 4. 設計標準

### 4.1 契約定義

- リソース
- メソッド
- パラメータ
- 応答コード
- エラーレスポンス
- 認証方式
- レート制御

### 4.2 応答設計

- 正常応答は一貫した構造とする
- エラー応答はコード、メッセージ、追跡 ID を含める
- 利用者が判断可能な粒度で返却する

### 4.3 バージョニング

- 互換性破壊を伴う変更はメジャーバージョン変更とする
- 旧版廃止時は通知期間を設ける
- バージョン混在期間を考慮した移行計画を作成する

## 5. セキュリティ標準

- 外部公開 API は認証必須とする
- 認証方式は原則 `OAuth` を優先する
- 必要に応じて `mTLS`、IP 制限、レート制御を組み合わせる
- バックエンド認証情報を API 利用者に露出しない

## 6. 運用標準

- API 利用状況を監視可能とする
- 呼出量、エラー率、応答時間を定常監視する
- 障害時に問い合わせ可能なオーナを定義する

## 7. 公開審査の必須項目

- 公開目的は明確か
- 利用者は特定されているか
- データ公開範囲は妥当か
- レート制御は定義済みか
- エラー応答は利用者にとって十分か
- バージョン管理方針は定義済みか

## 8. 禁止事項

- バックエンドの内部構造をそのまま露出する
- 認証なし公開
- 仕様書なし公開
- 廃止計画なしの breaking change
