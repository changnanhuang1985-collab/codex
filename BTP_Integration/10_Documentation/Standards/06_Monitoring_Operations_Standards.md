# 監視・運用標準

## 1. 目的

本書は本番稼働後の監視、障害一次対応、運用手順、変更管理の標準を定義する。

## 2. 基本方針

- 本番連携は監視前提で設計する
- 障害時に「誰が何を確認するか」を明確化する
- 一時回避と恒久対策を区別して管理する

## 3. 監視対象

- メッセージ処理失敗件数
- API エラー率
- 応答遅延
- キュー滞留
- 接続失敗
- 認証失敗

## 4. 運用標準

### 4.1 障害受付

- 発生時刻
- 対象連携
- 影響範囲
- 優先度
- 暫定対応状況

### 4.2 一次切り分け

- 再現有無
- 直近変更有無
- 認証異常か
- 接続異常か
- データ異常か
- 一時障害か恒常障害か

### 4.3 エスカレーション

- 業務影響大の場合は即時連絡
- 影響範囲未確定でも高リスクなら先に共有する
- 顧客報告と内部技術調査を並行実施する

## 5. 再処理標準

- 再処理可能条件を事前定義する
- 冪等性未確認のまま再送しない
- 手動再送の操作履歴を残す

## 6. 変更管理標準

- 本番変更は申請、承認、実施、結果記録を残す
- 緊急修正も事後記録を必須とする
- 変更前後の影響確認を行う

## 7. 運用ドキュメント必須項目

- 監視方法
- 異常時手順
- 連絡先
- 再処理手順
- ロールバック方針
- FAQ
