# 納品・ガバナンス標準

## 1. 目的

本書は、プロジェクトにおける成果物管理、承認、移行、顧客説明責任を標準化することを目的とする。

## 2. 基本方針

- 成果物はレビュー可能であること
- 設計、実装、試験、移行、運用の整合を確保すること
- 顧客説明可能な状態で納品すること

## 3. 成果物標準

最低限整備すべき成果物：

- 要件整理資料
- 連携設計書
- API / iFlow 一覧
- マッピング定義
- テスト結果
- 移行手順書
- 運用手順書
- 障害時対応手順

## 4. 承認管理

- 設計承認
- 開発完了承認
- テスト完了承認
- 本番移行承認

承認者、承認日、条件、保留事項を記録する。

## 5. 変更・課題管理

- 変更要求は影響評価を伴うこと
- 緊急変更も事後承認・記録を行うこと
- 未解決課題は一覧化し、責任者と期限を明確にすること

## 6. 顧客向け説明の観点

- 標準化された方式で設計していること
- セキュリティと運用を含めて考慮していること
- 拡張・保守を前提とした構造であること
- 障害時の対応責任が明確であること

## 7. 例外管理

- 標準からの逸脱は申請制とする
- 例外理由、影響、代替策、承認者を記録する
- 恒常的な例外は標準改訂対象として扱う
