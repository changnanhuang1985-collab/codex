# SAP BTP Integration 入门 Bootcamp

## 1. 文档定位

这是一套面向部门新人或跨岗同事的 `SAP BTP Integration` 入门级教育资料。

设计原则不是“快速讲完”，而是：
- 上手容易
- 持续有反馈
- 每一关都能通一点
- 但真正通关并不容易

也就是说，它更像一个训练游戏，而不是一份一次性看完的教材。

## 2. 三个角色如何协作设计这套培训

### PMO 视角：负责节奏与过关机制

- 把学习内容拆成短关卡
- 每关控制在 `30-90` 分钟内可完成
- 每周有固定节奏、复盘和提醒
- 每关必须有“看得见的产出”

### Senior Consultant 视角：确保不是只学技术

- 每一关都要回答一个业务问题
- 学员不能只知道“怎么配”，还要知道“为什么这样做”
- 尽早建立“业务场景 -> 集成模式 -> 平台能力”的思考框架

### Architect 视角：确保从一开始就不走歪

- 入门阶段先建立正确分层，而不是堆功能
- 学员要知道 `Cloud Integration`、`API Management`、`Event Mesh` 各自负责什么
- 从一开始就建立架构边界、治理意识和非功能意识

## 3. 游戏化设计原则

## 3.1 上手容易

第一天就能做出东西：
- 能登录学习平台
- 能申请试用环境
- 能打开 `Integration Suite`
- 能看懂一个最简单的 `iFlow`

这样学员会有“我能学会”的感觉。

## 3.2 通关不容易

真正难的是下面这些：
- 解释为什么选择某种集成模式
- 区分 `API-based`、`Process`、`Event-driven` 的适用场景
- 用自己的话讲清楚架构分层
- 从业务语言翻译到技术方案
- 独立完成一个可运行、可解释、可测试的小案例

所以课程要让学员不断获得小成功，但最终过关标准不能太松。

## 3.3 每关都要有三件事

- 一个明确任务
- 一个可验证产出
- 一个升级门槛

## 4. 建议学习对象

- 新加入 `SAP BTP Integration` 团队的同事
- 从 `ABAP`、`Basis`、`功能顾问` 转岗过来的成员
- 想从 `CPI Developer` 走向顾问或架构路线的人

## 5. 总体训练地图

| 阶段 | 主题 | 目标 | 建议时长 | 过关方式 |
| --- | --- | --- | --- | --- |
| Episode 0 | 开局与环境 | 能进入学习与练习环境 | 0.5 天 | 完成账号与环境准备 |
| Level 1 | 看懂 Integration Suite | 建立全景概念 | 0.5-1 天 | 口头讲清能力地图 |
| Level 2 | 第一个 iFlow | 完成最小可运行集成 | 1 天 | 成功部署并调用 |
| Level 3 | API 封装 | 知道如何把集成能力对外暴露 | 1 天 | API 可调用 |
| Level 4 | 监控与排错 | 学会最基础的运行问题定位 | 0.5-1 天 | 能解释报错与排查路径 |
| Level 5 | 模式判断 | 会选用基础集成模式 | 1 天 | 能说明方案取舍 |
| Boss 1 | 小项目实战 | 独立完成一个场景闭环 | 2-3 天 | 评审通过 |

## 6. 关卡设计

## Episode 0 - 开局：拿到装备

### 目标

建立最基本的学习条件和练习环境。

### 学员任务

- 注册 `SAP BTP Trial`
- 进入 `SAP Integration Suite Trial`
- 了解学习平台入口
- 加入社区入口

### 最低产出

- 一张能证明已经进入 `Integration Suite` 的截图
- 一张个人学习清单

### 过关标准

- 能说清楚学习入口在哪里
- 知道试用环境不是生产环境

### PMO 提醒

- 这一关必须当天过掉
- 没有环境就没有后续训练

## Level 1 - 世界观建立：先知道自己在学什么

### 目标

建立 `Integration Suite` 的整体认知，不要求马上能开发，但要求知道各能力是干什么的。

### 学员必须回答的问题

- `Cloud Integration` 是做什么的
- `API Management` 是做什么的
- `Event Mesh` 是做什么的
- `Integration Advisor` 适合什么场景
- `Open Connectors` 的价值是什么

### 推荐输出

- 1 页能力地图
- 5 分钟口头介绍

### 升级门槛

不能只会背名词，必须能把“能力”和“场景”对应起来。

## Level 2 - 第一战：做出第一个 iFlow

### 目标

完成一个最小可运行的 `iFlow`。

### 任务设计

建议先做官方最基础的练习：
- 创建 `Integration Package`
- 创建并部署第一个 `iFlow`
- 通过 `OData` 或 `HTTP` 获取数据

### 学员会获得的成就感

- 第一次真正“跑起来”
- 第一次看到集成流转

### 真正的难点

- 不是拖几个图形，而是理解消息怎么流动
- 不是部署成功，而是能说清楚每一步在做什么

## Level 3 - 第二战：把 iFlow 包成 API

### 目标

理解为什么很多集成能力最终要通过 `API` 对外暴露。

### 任务设计

- 使用 `API Management` 暴露 `iFlow`
- 做一次简单认证配置
- 完成一次调用验证

### Senior Consultant 要强调的点

- 这不是“技术炫技”
- 而是把后端能力产品化、可治理化

### Architect 要强调的点

- `API Layer` 和 `Integration Layer` 不应混为一谈
- 暴露能力和编排逻辑是两件相关但不同的事

## Level 4 - 第三战：学会看监控，不做“只会开发不会排错”的人

### 目标

掌握最基础的运行监控与问题排查。

### 任务设计

- 进入 `Message Monitoring`
- 看 `Trace`
- 看 `MPL Logs`
- 故意制造一个简单错误并定位

### 升级门槛

能回答：
- 错在哪里
- 为什么错
- 下一步该看哪里

## Level 5 - 第四战：开始做判断，而不是只做操作

### 目标

建立“模式选择”意识。

### 训练题

给学员几个业务场景，让他判断更适合哪种模式：
- `Process Integration`
- `Data Integration`
- `API-based Integration`
- `Event-driven Integration`

### 示例场景

- `Employee Onboarding`
- `Business Partner Replication`
- `SalesOrder.Created`
- 第三方系统调用主数据查询接口

### 过关标准

不只给答案，还要说清原因。

## Boss 1 - 小项目实战

### 任务目标

独立完成一个“小而完整”的训练场景。

### 推荐场景

- `File -> SAP`
- `SuccessFactors -> S/4` 的简化版
- `Public API -> iFlow -> API Proxy`

### 交付物

- 场景说明
- 简单架构图
- `iFlow` 设计说明
- 调用结果
- 问题与排查记录

### 通关标准

- 能跑
- 能讲
- 能解释为什么这么设计

## 7. 评分机制建议

建议采用 100 分制：

| 项目 | 分值 | 说明 |
| --- | ---: | --- |
| 完成度 | 30 | 是否真的做出来 |
| 理解度 | 25 | 是否讲得清楚 |
| 业务感 | 15 | 是否理解业务目的 |
| 架构感 | 15 | 是否知道边界与分层 |
| 排错能力 | 15 | 是否具备基本问题定位能力 |

建议把“会做”与“会讲”分开评分。

## 8. 培训节奏建议

### 第 1 周

- 完成 `Episode 0`
- 完成 `Level 1`
- 看完官方入门学习路径前两单元

### 第 2 周

- 完成 `Level 2`
- 完成 `Level 4`
- 提交第一份 `iFlow` 说明

### 第 3 周

- 完成 `Level 3`
- 完成 `Level 5`
- 做一次模式选择练习

### 第 4 周

- 完成 `Boss 1`
- 做 15-20 分钟结业汇报

## 9. 讲师分工建议

### PMO

- 维护学习日历
- 每周提醒
- 跟踪是否过关
- 整理问题清单

### Senior Consultant

- 解释每一关的业务意义
- 把技术任务翻译成客户价值
- 负责结业汇报口径

### Architect

- 审看是否建立了正确架构认知
- 判断是否存在“会配不会设计”的问题
- 在 `Boss 1` 阶段做架构评审

请配合本目录中的：

- `SAP_Integration_Official_Resources.md`

建议使用方式：
- `Episode 0` 对应试用环境和账号准备
- `Level 1` 对应 `SAP Learning Journey`
- `Level 2` 对应第一个 `iFlow` 教程

## 11. 最后原则

这套培训不是为了“看起来学过”，而是为了让新人逐步具备下面三种能力：

- 能做一个小集成
- 能说清楚为什么这样做
- 能逐步走向顾问和架构视角

如果只会点界面，不算通关。
