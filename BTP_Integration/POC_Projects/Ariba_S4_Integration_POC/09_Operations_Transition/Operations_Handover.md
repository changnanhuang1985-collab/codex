# 運用引継ぎ資料

## 目的

POC 完了後に、プロジェクトチームから運用 / 支援側へ引き継ぐべき最小限の内容を整理する。

## 引継ぎ項目

- IF 一覧および目的
- Endpoint / 設定値一覧
- 監視ポイント
- エラー区分と一次切り分け方針
- 再処理可否および注意点
- 既知制約および未解決事項

## 初期運用向け注意事項

- Trace は調査時のみ有効化する
- MPL 検索キーを運用担当へ共有する
- Value Mapping 変更責任を実案件開始前に明確化する

## 次工程に向けた推奨事項

- 本番運用 Runbook を別途作成する
- 障害優先度マトリクスを定義する
- Integration と業務アプリ側の連絡導線を明確化する
