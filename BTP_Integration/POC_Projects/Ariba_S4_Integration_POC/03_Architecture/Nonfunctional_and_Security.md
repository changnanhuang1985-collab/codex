# 非機能要件およびセキュリティ観点

## 非機能要件

- 過度な最適化よりも設計の明確性を優先する
- 業務キーおよびメッセージ ID による追跡性を確保する
- 例外を種別ごとに判別できるようにする
- 将来案件で流用できる連携パターンとする

## セキュリティ観点

- エンドポイント認証方式は設計レベルで明示する
- OAuth または証明書接続の前提を文書化する
- 設計書および開発成果物に平文認証情報を残さない
- 環境依存情報は外部化する

## 運用観点

- エラー発生時に CPI Monitoring で追跡できること
- 障害を原因種別ごとに切り分けられること
- 最低限の運用引継ぎ情報を準備すること
