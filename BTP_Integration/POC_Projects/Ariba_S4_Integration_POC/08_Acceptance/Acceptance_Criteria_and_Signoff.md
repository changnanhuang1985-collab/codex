# 受入基準およびサインオフ

## 受入基準

- 必要なプロジェクト成果物が納品されていること
- アーキテクチャおよび IF 設計が内部レビュー済みであること
- 主要 CPI シナリオが説明可能であること
- 試験ケースおよび試験結果サマリが揃っていること
- 課題および前提条件が明示されていること
- 次回案件向けテンプレートとして再利用価値があること

## 顧客確認観点

- POC の対象範囲が現実的で理解しやすいか
- CPI 中心のアーキテクチャが今後の実案件方針と整合しているか
- 成果物が次フェーズ案件で再利用できるか
