# 质量控制与 Review 设计

## 1. 总体原则

本工具的第一优先级是正确性，不是自动化率。

因此必须建立：

- 自动检查
- checklist
- reviewer 复核
- 变更回归检查

的四层控制。

## 2. 自动检查项建议

- 关键字段是否缺失
- 文件命名是否符合规则
- 必填动态字段是否全部落位
- 固定文本是否被异常修改
- 模板版本与规则版本是否匹配

## 3. 人工 Review 重点

- 高风险条款变量
- 日期、金额、客户名、项目编号
- 特殊条件块是否正确触发
- 多份输出文档间是否一致
- 新模板或新规则是否经过确认

## 4. Checklist 建议结构

- 输入文件完整性检查
- 动态字段提取检查
- 模板匹配检查
- 输出文件命名检查
- 关键字段一致性检查
- reviewer 签字确认

## 5. 升级与回归控制

当模板、规则或输入格式发生变化时，必须：

1. 更新变更记录
2. 标识影响范围
3. 准备回归样本
4. 重新执行 checklist
5. 完成 review 后再正式采用

## 6. 结论

本工具应采用“工具生成 + 检查 + review + 回归”的模式，不能把正确性寄托在单次生成结果的直觉判断上。
