# 工具概念与工作流设计

## 1. 工具定位

该工具建议定位为：

`规则驱动的文档转换与模板生成工具`

而不是单纯的“AI 自由生成工具”。

## 2. 核心工作流

建议工作流如下：

1. 导入客户 `input`
2. 解析输入内容
3. 提取动态字段
4. 关联目标模板
5. 应用转换规则
6. 生成输出文档和文件名
7. 执行自动检查
8. 输出 review checklist
9. 由 reviewer 进行确认
10. 归档模板、规则和版本信息

## 3. 关键设计思想

### 3.1 模板优先

输出文档应优先基于模板生成，而不是每次从零生成。

### 3.2 规则显式化

所有转换规则应写成显式对象，而不是隐含在 prompt 或人工习惯里。

### 3.3 人工 review 不可省

高风险契约文档不适合完全取消人工 review。

### 3.4 变更可升级

模板、规则、命名、字段抽取都应有升级机制。

## 4. 建议产物分层

- `Input Profile`
- `Field Extraction Rule`
- `Template File`
- `Transformation Rule`
- `Filename Rule`
- `Checklist Rule`
- `Review Record`

## 5. 后续可技术化方向

- 模板引擎
- 规则配置文件
- 输入字段抽取器
- 输出差异检查器
- 版本管理与回归检查机制
