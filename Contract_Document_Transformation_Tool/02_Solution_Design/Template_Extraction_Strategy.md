# 模板提炼策略

## 1. 为什么模板提炼是核心

这个工具的关键不是“生成文字”，而是先把输出文档拆成：

- 固定文字
- 动态字段
- 条件块
- 文件命名变量

只有模板提炼做好，后续自动生成才稳定。

## 2. 建议识别维度

### 2.1 固定文字

在不同项目中基本不变化，应固化为模板正文。

### 2.2 动态字段

会随客户输入变化，需要明确字段来源、格式和填充位置。

### 2.3 条件块

某些条款、说明、页眉页脚、附加说明可能只在特定条件下出现。

### 2.4 命名变量

输出文件名中的客户名、项目编号、日期、版本号等，也应视为模板变量。

## 3. 模板提炼建议流程

1. 收集多组 input / output 样本
2. 做样本对比，区分固定与动态部分
3. 归纳动态字段字典
4. 归纳条件分支
5. 定义模板变量命名
6. 形成模板文件与字段映射表

## 4. 建议输出资产

- 模板主文件
- 字段字典
- 条件规则清单
- 文件命名规则表
- 样本对比记录

## 5. 设计原则

- 模板变量命名要稳定
- 固定文字不得混入动态规则
- 动态字段必须可追溯到输入来源
- 每次模板更新都要保留版本说明
