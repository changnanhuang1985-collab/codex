# Testing And Review Strategy

## 1. 目的

本文档定义本项目第一版测试与 review 策略。

## 2. 测试目标

- 验证字段识别是否正确
- 验证模板变量是否完整
- 验证文件命名是否符合规则
- 验证固定文字没有被误改
- 验证不同输出类别是否选择了正确模板

## 3. 测试分层

### 3.1 样例治理测试

- 样例是否正确登记
- 样例是否标识为真实 input 或参考 output
- 样例关系说明是否完整

### 3.2 字段测试

- 客户名称提取正确
- 项目主题提取正确
- 编号提取正确
- 无法识别项是否被标记

### 3.3 模板测试

- 固定标题是否正确
- 动态变量占位是否完整
- 参考模板中的历史项目特有字段是否被隔离

### 3.4 命名测试

- 文件名变量顺序是否正确
- 文件名中的客户名和项目名是否与正文一致
- 是否存在使用历史项目名的误生成

### 3.5 Review 测试

- 高风险字段是否进入 reviewer 视野
- reviewer 是否有可操作的检查清单
- 规则变化后是否要求回归检查

## 4. Reviewer 重点看什么

- 客户名称
- 项目主题
- 项目编号 / 见积编号
- 日期或期间
- 内部文书类别是否选对
- 固定文案是否沿用了错误的历史项目信息

## 5. 当前阶段可执行的检查结论

当前阶段已经可以做：

- 样例治理检查
- 文件名级别检查
- 字段字典草案检查
- 参考模板拆解检查

当前阶段还不能完整做：

- 正文级自动抽取正确性验证
- 最终输出内容逐字段比对

## 6. 结论

测试应跟着项目建模阶段走，先把“样例对不对、变量对不对、模板边界对不对”检查好，再进入正式自动转换测试。
