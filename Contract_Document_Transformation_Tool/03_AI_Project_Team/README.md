# Contract Tool AI 项目团队

## 1. 团队目标

本 AI 项目团队用于支撑 `Contract Document Transformation Tool` 的规划、设计、实现、治理与持续升级。

这不是普通写文档小组，而是一支围绕“高严谨度文档转换工具”进行协作的专门队伍。

## 2. 角色列表

- `AI_01_Contract_Product_Owner.md`
- `AI_02_Template_and_Rule_Architect.md`
- `AI_03_Extraction_and_Transformation_Engineer.md`
- `AI_04_Quality_and_Review_Lead.md`
- `AI_05_Change_Learning_and_Operations_Lead.md`

## 3. 推荐协作顺序

1. `Contract Product Owner` 明确业务目标、样本范围和输出要求
2. `Template and Rule Architect` 拆分模板、规则、字段和命名逻辑
3. `Extraction and Transformation Engineer` 规划工具实现与转换引擎
4. `Quality and Review Lead` 设计 checklist、review 和回归机制
5. `Change Learning and Operations Lead` 负责格式变更、规则升级和持续运营

## 4. 与现有核心角色的关系

- `大黄`：负责统筹与推进
- `Kai`：负责版本治理与提交纪律
- `Nova`：负责 Agent workflow 与 AI 工作方式升级
- `Orion`：负责 changnan_codex 级别的全局管理与升级判断

## 5. 一句话说明

这个 AI 团队的目标，是把“容易出错的手工契约文档转换”升级成“模板化、规则化、可检查、可持续演进的工作系统”。
